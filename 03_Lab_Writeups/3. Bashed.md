# HTB 实战复盘笔记：Bashed (IP: 10.129.34.197)

## [Executive Summary | 执行摘要]

Bashed 是一台典型的 **Web 配置疏忽**驱动的靶机。攻击链从 Web 目录泄露（phpbash）开始，经过 Sudoers 权限配置错误实现横向移动，最终利用 **Root 级定时任务（Cron Job）与文件权限覆盖**缺陷，完成从低权限 Web 用户到系统最高权限的垂直跃迁。

---

## 1. 核心信息搜集 (Enumeration)

* **常规探测**：
* `nmap -sC -sV -Pn 10.129.34.197`
* **结果**：仅开放 `80/tcp` (Apache 2.4.18)。确定战场全线集中在 Web 端。


* **Web 目录爆破 (Gobuster)**：
* `gobuster dir -u http://10.129.34.197 -w /usr/share/wordlists/dirb/common.txt`
* **深度发现**：扫出 `/dev/` 目录。该目录通常用于开发环境，在生产服务器中出现属于严重配置失误。


* **注意什么（实战避坑指南）**：
* **目录扫描的广度**：若 `common.txt` 未果，应立刻更换 `SecLists/Discovery/Web-Content/directory-list-2.3-medium.txt`。
* **环境识别**：看到 `/dev` 或 `/uploads` 目录时，应立刻联想“文件上传”或“开发者残留工具”攻击向量。



---

## 2. 漏洞分析与初始打点 (Initial Foothold)

* **漏洞原理**：隐蔽式安全（Security by Obscurity）失效。开发者将功能强大的 Web Shell (`phpbash.php`) 直接留在 Web 可访问目录下，未做任何身份认证。
* **操作流**：
1. 访问 `http://10.129.34.197/dev/phpbash.php`。
2. 直接获取交互式网页终端，当前权限：`www-data`。


* **Payload 说明**：
* `phpbash` 本质上是一个封装了 PHP `exec()` 或 `shell_exec()` 函数的脚本，通过 HTTP POST 请求将命令传递给底层操作系统。



---

## 3. 横向移动：权限滥用 (Horizontal PrivEsc)

* **Sudo 配置枚举**：
* `sudo -l`
* **核心发现**：`(scriptmanager : scriptmanager) NOPASSWD: ALL`。


* **逻辑深挖（为什么）**：
* 系统管理员为了方便开发，配置了 `sudoers` 文件，允许 Web 用户 `www-data` 无需密码即可切换至 `scriptmanager` 用户。


* **提权指令**：
* `sudo -u scriptmanager bash` (在 `phpbash` 环境下可能因缺少 TTY 报错，建议直接带参数执行后续操作)。



---

## 4. 垂直提权：定时任务劫持 (Vertical PrivEsc)

这是本机的核心提权逻辑，涉及 **Linux 计划任务 (Cron Job)** 与 **文件属主权限** 的逻辑错位。

### A. 环境侦察 (Situational Awareness)

* 执行 `ls -la /` 发现非标准根目录文件夹 `/scripts`，其属主为 `scriptmanager`。
* 执行 `ls -la /scripts`：
* `test.py` (Owner: **scriptmanager**) -> 你可以修改。
* `test.txt` (Owner: **root**) -> 只有 Root 能创建。


* **底层机制推演**：通过观察 `test.txt` 的时间戳每分钟自动更新，判定系统后台存在一个以 **Root 权限** 运行的定时任务，该任务逻辑为：`python /scripts/test.py`。

### B. 漏洞利用 (Exploitation)

由于 `root` 会盲目执行 `test.py`，而该文件对 `scriptmanager` 开放写权限，我们可以实施**代码注入**。

* **Payload 选择（Python 反向 Shell 逻辑）**：
为了避开 Web Shell 的引号解析陷阱，使用 `tee` 命令进行覆盖：
```bash
echo 'import os; os.system("cp /root/root.txt /tmp/root.txt; chmod 777 /tmp/root.txt")' | sudo -u scriptmanager tee /scripts/test.py

```


* **代码原理解析**：
* `import os; os.system(...)`：利用 Python 的 OS 模块调用底层 Shell 指令。
* `cp /root/root.txt /tmp/root.txt`：强制 Root 帮你把受保护的 Flag 复制到公共临时目录。
* `chmod 777`：确保复制出来的文件全员可读。



### C. 获取上帝权限 (The Final Push)

* **等待触发**：Cron 任务通常每一分钟（第 0 秒）触发一次。
* **提货指令**：`cat /tmp/root.txt`。

## 5. 总结与 Checklist (Review)

| 步骤 | 关键命令 / Payload | 实战避坑建议 |
| --- | --- | --- |
| **目录扫描** | `gobuster dir -u <URL> -w <Wordlist>` | 漏写 `dirb` 路径中的 `b` 会导致扫描失败。 |
| **权限检查** | `sudo -l` | 必做项。优先寻找 `NOPASSWD` 标记。 |
| **定时任务排查** | `ls -la /scripts` (观察时间戳动态) | 若无法执行 `crontab -l`，通过文件时间戳变化推断任务是唯一路径。 |
| **代码注入** | `echo 'payload' | sudo -u  tee ` |
| **Flag 提取** | 浏览器开发者工具 (F12) 审查元素 | 当 Web Shell UI 渲染截断长字符串时，直接看 DOM 源码是红队必备的“降维打击”思维。 |

### [附录：常用路径速查]

* **Web 路径**：`/var/www/html/`
* **用户家目录**：`/home/<user>/`
* **系统二进制文件**：`/usr/bin/` (理解 `bin` = Binary，即命令存放地)
* **临时目录**：`/tmp/` (提权过程中转文件的首选地)
